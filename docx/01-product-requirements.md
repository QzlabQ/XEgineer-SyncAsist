# 产品需求文档（PRD）

## 1. 背景与问题

内容创作者通常需要在多个平台同步发布内容（微信公众号、知乎、B站、小红书等），面临以下痛点：

- **格式繁琐**：各平台编辑器格式不统一，同一篇文章需要反复排版
- **效率低下**：逐平台手动复制粘贴，图片需重复上传
- **体验割裂**：在多个平台编辑器之间切换，心智负担重
- **格式损失**：复制粘贴常导致格式丢失，需要二次修复

---

## 2. 目标用户

| 用户类型 | 特征 | 核心诉求 |
|----------|------|----------|
| 技术博主 | 在掘金、CSDN、知乎、公众号同步发布 | 代码块格式保留，一键同步 |
| 自媒体创作者 | 在公众号、小红书、微博同步发布 | 图文排版，封面管理 |
| 知识付费作者 | 在知乎、B站专栏、公众号发布 | 内容质量，格式精准 |
| 企业内容团队 | 多人协作，多平台矩阵运营 | 团队协作，发布审批 |

## 3. 功能需求

### 3.1 编辑器（P0）

- 所见即所得富文本编辑，支持：
  - 多级标题（H1-H4）
  - 加粗、斜体、下划线、删除线、行内代码
  - 有序列表、无序列表、任务列表
  - 代码块（带语言高亮）
  - 引用块
  - 图片（拖拽上传、粘贴上传、URL 插入）
  - 表格
  - 分割线
  - 超链接
- 文章标题单独输入框（大字体，突出）
- 自动保存（本地 IndexedDB，间隔 3 秒）
- 文章列表管理（新建、删除、重命名）

### 3.2 平台预览（P0）

- 编辑区右侧展示多平台预览面板
- Tab 切换查看不同平台的渲染效果
- 预览实时更新（debounce 300ms）
- 支持平台：微信公众号、知乎、B站专栏、小红书、掘金

### 3.3 平台配置（P0）

- 每个平台可独立配置：
  - 封面图（上传或从正文图片中选取）
  - 摘要（自动截取前 100 字或手动填写）
  - 标签 / 话题
  - 分类
  - 是否保存为草稿（默认开启）
- 配置信息随文章保存，下次打开自动恢复

### 3.4 一键发布（P0）

- 勾选目标平台后点击"发布"
- 通过 Chrome 扩展调用各平台 API（使用浏览器已有 Cookie）
- 实时展示各平台发布进度与结果
- 发布成功后展示文章链接 / 草稿链接
- 支持重试失败的平台

### 3.5 账号管理（P1）

- 展示各平台登录状态（已登录 / 未登录）
- 未登录时引导用户在浏览器中登录对应平台
- 支持刷新登录状态检测

### 3.6 发布历史（P1）

- 记录每篇文章的发布记录（平台、时间、链接、状态）
- 支持查看历史发布链接

### 3.7 图片 CDN（P2）

- 统一上传图片到自有 CDN
- 避免各平台图片外链失效问题
- 发布时自动替换图片 URL

### 3.8 AI 辅助（P2）

- 自动生成文章摘要
- 自动推荐标签
- 自动生成封面图（AI 绘图）

---

## 4. 非功能需求

| 指标 | 要求 |
|------|------|
| 编辑器响应 | 输入延迟 < 16ms（60fps） |
| 预览更新 | debounce 后 < 500ms 完成渲染 |
| 发布单平台 | < 10s（网络正常情况下） |
| 本地存储 | 支持 1000+ 篇文章，不影响性能 |
| 浏览器兼容 | Chrome 120+（扩展依赖） |

---

## 5. 约束条件

- **必须使用 Chrome 扩展**：各平台 API 需要携带浏览器 Cookie，无法从服务端直接调用
- **不存储用户凭证**：平台 Cookie 仅在浏览器本地使用，不上传服务端
- **默认草稿模式**：发布默认保存为草稿，用户手动确认后正式发布（降低误操作风险）
