# 开发路线图

---

## Phase 1：MVP（预计 4-6 周）

**目标**：跑通核心链路——编辑 → 预览 → 发布到知乎/B站

### Week 1-2：基础框架搭建

- [ ] 初始化 Monorepo（Yarn Workspaces + TypeScript）
- [ ] 将 Wechatsync `packages/core` 集成为子包
- [ ] 搭建 Next.js 15 应用骨架（路由、布局）
- [ ] 集成 Tiptap v2，实现基础富文本编辑
- [ ] 实现 `packages/renderer` 包骨架（ContentAST 类型定义）
- [ ] 实现 Tiptap JSON → ContentAST 转换

### Week 3-4：平台渲染器 + 预览

- [ ] 实现知乎 Renderer（HTML 格式）
- [ ] 实现 B站 Renderer（HTML 格式）
- [ ] 实现掘金 Renderer（Markdown 格式）
- [ ] 实现三栏布局 UI（编辑区 + 预览区 + 左侧面板）
- [ ] 实现多平台预览 Tab 切换
- [ ] 实现预览实时更新（debounce）

### Week 5-6：扩展桥接 + 发布链路

- [ ] 实现 Chrome Extension 桥接层（Content Script + 消息协议）
- [ ] 集成 Wechatsync 平台适配器到 Service Worker
- [ ] 实现发布控制台 UI（平台选择、进度展示、结果展示）
- [ ] 实现本地文章存储（Dexie.js）
- [ ] 实现自动保存
- [ ] 端到端测试：编辑 → 发布到知乎草稿

**MVP 交付物**：
- 可用的 Web 编辑器
- 知乎、B站、掘金三个平台的预览和发布
- Chrome 扩展（本地安装）
- 本地文章存储

## Phase 2：完善（预计 4-6 周）

**目标**：覆盖主流平台，完善用户体验

### 平台扩展

- [ ] 微信公众号 Renderer（内联样式 HTML）
- [ ] 小红书 Renderer（图文结构）

- [ ] 简书 Renderer（HTML）

### 功能完善

- [ ] 平台专属配置面板（封面图、摘要、标签）

- [ ] 摘要自动截取
- [ ] 账号管理页面（登录状态检测）
- [ ] 发布历史记录
- [ ] 图片拖拽上传、粘贴上传

### 体验优化

- [ ] 编辑器工具栏优化（悬浮工具栏）
- [ ] 键盘快捷键
- [ ] 发布失败重试
- [ ] 扩展安装引导页面

## Phase 3：增强（持续迭代）

**目标**：差异化功能，提升竞争力

### AI 辅助

- [ ] 自动生成文章摘要（调用 Claude API）
- [ ] 自动推荐标签
- [ ] 标题优化建议
- [ ] 自动生成封面图（AI 绘图）

- [ ] 统一图片上传到自有 CDN

### 定时发布

- [ ] 设置发布时间

### 数据统计

- [ ] 各平台阅读量汇总展示

- [ ] 发布审批流程
- [ ] 团队账号管理

## 技术债务管理

以下事项在 MVP 阶段暂缓，后续迭代处理：

- 服务端存储（MVP 使用 IndexedDB 本地存储）
- 图片 CDN（MVP 直接使用原始 URL）
- 单元测试覆盖（MVP 阶段以手动测试为主）
- 错误监控（Sentry 等）
- 性能优化（代码分割、懒加载）
