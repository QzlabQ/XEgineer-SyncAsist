# 开发路线图

---

## Phase 1：MVP（预计 4-6 周）

**目标**：跑通核心链路——编辑 → 预览 → 发布到知乎/B站

### Week 1-2：基础框架搭建

- [x] 初始化 Monorepo（Yarn Workspaces + TypeScript）
- [x] 将 Wechatsync `packages/core` 集成为子包
- [x] 搭建 Next.js 15 应用骨架（路由、布局）
- [x] 集成 Tiptap v2，实现基础富文本编辑
- [x] 实现 `packages/renderer` 包骨架（ContentAST 类型定义）
- [x] 实现 Tiptap JSON → ContentAST 转换

### Week 3-4：平台渲染器 + 预览

- [x] 实现知乎 Renderer（HTML 格式）
- [x] 实现 B站 Renderer（HTML 格式）
- [x] 实现掘金 Renderer（Markdown 格式）
- [x] 实现三栏布局 UI（编辑区 + 预览区 + 左侧面板）
- [x] 实现多平台预览 Tab 切换
- [x] 实现预览实时更新（debounce）

### Week 5-6：扩展桥接 + 发布链路

- [x] 实现 Chrome Extension 桥接层（Content Script + 消息协议）
- [x] 集成 Wechatsync 平台适配器到 Service Worker
- [x] 实现发布控制台 UI（平台选择、进度展示、结果展示）
- [x] 实现本地文章存储（Dexie.js）
- [x] 实现自动保存
- [x] 端到端测试：编辑 → 发布到知乎草稿

**MVP 交付物**：

- 可用的 Web 编辑器
- 知乎、B站、掘金三个平台的预览和发布
- Chrome 扩展（本地安装）
- 本地文章存储

**目标**：覆盖主流平台，完善用户体验

**说明**：以下条目按当前代码实现状态更新；少数平台发布链路仍保留回退和人工确认。

### 功能完善

- [x] 平台专属配置面板（封面图、摘要、标签）
- [x] 封面图上传与管理（）
- [ ] 摘要自动截取(unfinish)
- [x] 账号管理页面（平台登录状态检测）
- [x] 发布历史记录
- [x] 图片拖拽上传、粘贴上传
- [x] `/` 命令菜单（快速插入块）

### 体验优化

- [x] 编辑器工具栏优化（悬浮工具栏）
- [x] 键盘快捷键
- [x] 发布失败重试
- [x] 扩展安装引导页面

### 真实环境验收备注

- 小红书发布链路采用页面自动化，不是纯 API 直发。
- 知乎封面写入为 best effort，失败时会回退为无封面草稿。
- 知乎“发布已有草稿”采用编辑页自动化点击发布，只有确认进入公开文章页才标记为已发布。
- 各平台真实账号 smoke test 仍在持续补充。
