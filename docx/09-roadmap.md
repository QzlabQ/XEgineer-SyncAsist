# 开发路线图

---

## Phase 1：MVP（预计 4-6 周）

**目标**：跑通核心链路——编辑 → 预览 → 发布到知乎/B站

### Week 1-2：基础框架搭建

- [x] 初始化 Monorepo（Yarn Workspaces + TypeScript）
- [x] 集成平台适配框架（BaseAdapter/CodeAdapter/RuntimeInterface）
- [x] 搭建 Next.js 15 应用骨架（路由、布局）
- [x] 集成 Tiptap v2，实现基础富文本编辑
- [x] 实现 `packages/renderer` 包骨架（ContentAST 类型定义）
- [x] 实现 Tiptap JSON → ContentAST 转换

### Week 3-4：平台渲染器 + 预览

- [x] 实现知乎 Renderer（HTML 格式）
- [x] 实现 B站 Renderer（HTML 格式）
- [x] 实现掘金 Renderer（Markdown 格式）
- [x] 实现三栏布局 UI（编辑区 + 预览区 + 左侧面板）
- [x] 实现多平台预览 Tab 切换
- [x] 实现预览实时更新（debounce）

### Week 5-6：扩展桥接 + 发布链路

- [x] 实现 Chrome Extension 桥接层（Content Script + 消息协议）
- [x] 集成平台适配器到 Service Worker（知乎、B站、掘金、微信公众号、CSDN）
- [x] 实现发布控制台 UI（平台选择、进度展示、结果展示）
- [x] 实现本地文章存储（Dexie.js）
- [x] 实现自动保存
- [x] 端到端测试：编辑 → 发布到知乎草稿

**MVP 交付物**：

- 可用的 Web 编辑器
- 知乎、B站、掘金三个平台的预览和发布
- Chrome 扩展（本地安装）
- 本地文章存储

## Phase 3：平台化与 AI（预计 8-12 周）

**目标**：从单机工具升级为多用户平台，引入 AI 辅助，打磨平台模板

---

### 3.1 平台化与数据持久化

**背景**：当前所有数据存储在浏览器 IndexedDB 中，单用户单设备使用。需要引入服务端实现多用户、跨设备、团队协作。

#### 用户系统

- [ ] 注册 / 登录 / 密码找回
- [ ] OAuth 登录（GitHub / Google）
- [ ] JWT 鉴权 + 刷新 Token
- [ ] 文章云端存储（PostgreSQL / Drizzle ORM）
- [ ] 图片上传到自有 CDN（OSS / S3）
- [ ] 发布历史云同步

#### 权限与协作

- [ ] 角色权限模型（Owner / Editor / Viewer）
- [ ] 文章分享链接（只读 / 可评论）
- [ ] 多人协同编辑（Yjs + WebSocket）
- [ ] 操作日志与版本历史
- [ ] 发布审批流程
- [ ] 团队空间（Team Space），多项目管理

#### 后端技术栈

- [ ] Next.js API Routes / tRPC
- [ ] PostgreSQL + Drizzle ORM
- [ ] Redis（会话 / 实时协作）
- [ ] 图片 CDN（OSS / S3 + 缩略图处理）

---

### 3.2 针对性模板优化

**背景**：当前各平台使用通用 HTML/Markdown 转换，不同平台的展示效果差异较大。需要针对各平台做 CSS 样式增强和模板定制。

#### 微信公众号 H5 渲染增强

- [ ] 段落间距、字体、行高自动适配微信客户端
- [ ] 代码块语法高亮渲染为图片（微信不支持 `<pre>` 样式）
- [ ] 表格自适应（微信窄屏适配）
- [ ] 引用块、列表的微信专属样式
- [ ] 封面图自动裁剪为微信尺寸（900×500 / 2.35:1）
- [ ] 正文图片自动转存微信 CDN + 替换 URL

#### 小红书图文模板

- [ ] 多图排版模板（4 图 / 6 图 / 9 图布局）
- [ ] 正文分段 + emoji 点缀自动生成
- [ ] 话题标签智能推荐与排序
- [ ] 封面图自动裁剪为小红书尺寸（3:4）
- [ ] 长文自动拆分多篇笔记

#### 通用模板系统

- [ ] 平台模板 DSL（JSON Schema 定义模板参数）
- [ ] 模板市场（预设模板库，一键应用）
- [ ] 品牌自定义模板（Logo、配色、字体）
- [ ] 模板预览（在编辑器中实时切换模板查看效果）
- [ ] CSS 样式引擎（从 Markdown AST 生成内联样式 HTML，对标微信公众号编辑器）

---

### 3.3 AI 功能

**背景**：利用 LLM 能力辅助内容创作，降低创作者成本。

#### AI 辅助写作

- [ ] 文章摘要自动生成（调用 Claude API）
- [ ] 标题优化建议（生成 3-5 个备选标题）
- [ ] 自动推荐标签（基于文章内容语义分析）
- [ ] 智能配图推荐（根据段落内容推荐图片）
- [ ] 一键改写（调整语气：专业 / 轻松 / 小红书风格）
- [ ] 大纲生成（输入主题 → 输出结构化大纲）
- [ ] 续写 / 扩写 / 缩写

#### AI 图片

- [ ] AI 封面图生成（DALL·E / Stable Diffusion）
- [ ] 正文插图生成（根据段落内容自动生成配图）
- [ ] 图片背景移除 / 智能裁剪

#### AI 平台适配

- [ ] 一键平台风格转换（同一篇文章 → 知乎严肃风 / 小红书轻松风 / 微信公众号深度风）
- [ ] 平台规范检查（标题长度、图片数量、敏感词）
- [ ] SEO 优化建议（标题关键词、标签策略）
