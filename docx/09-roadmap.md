# 开发路线图

---

## Phase 1：MVP（预计 4-6 周）

**目标**：跑通核心链路——编辑 → 预览 → 发布到知乎/B站

### Week 1-2：基础框架搭建

- [x] 初始化 Monorepo（Yarn Workspaces + TypeScript）
- [x] 集成平台适配框架（BaseAdapter/CodeAdapter/RuntimeInterface）
- [x] 搭建 Next.js 15 应用骨架（路由、布局）
- [x] 集成 Tiptap v2，实现基础富文本编辑
- [x] 实现 `packages/renderer` 包骨架（ContentAST 类型定义）
- [x] 实现 Tiptap JSON → ContentAST 转换

### Week 3-4：平台渲染器 + 预览

- [x] 实现知乎 Renderer（HTML 格式）
- [x] 实现 B站 Renderer（HTML 格式）
- [x] 实现掘金 Renderer（Markdown 格式）
- [x] 实现三栏布局 UI（编辑区 + 预览区 + 左侧面板）
- [x] 实现多平台预览 Tab 切换
- [x] 实现预览实时更新（debounce）

### Week 5-6：扩展桥接 + 发布链路

- [x] 实现 Chrome Extension 桥接层（Content Script + 消息协议）
- [x] 集成平台适配器到 Service Worker（知乎、B站、掘金、微信公众号、CSDN）
- [x] 实现发布控制台 UI（平台选择、进度展示、结果展示）
- [x] 实现本地文章存储（Dexie.js）
- [x] 实现自动保存
- [x] 端到端测试：编辑 → 发布到知乎草稿

**MVP 交付物**：

- 可用的 Web 编辑器
- 知乎、B站、掘金三个平台的预览和发布
- Chrome 扩展（本地安装）
- 本地文章存储

## Phase 3：平台化（预计 4-6 周）

**目标**：服务端存储 + 用户系统，支持多用户协作

### Week 1-2：用户系统

- [ ] 注册 / 登录 / 找回密码
- [ ] JWT 鉴权 + Token 刷新
- [ ] 文章云端存储（替换 IndexedDB 本地存储）
- [ ] 发布历史云同步

### Week 3-4：协作与权限

- [ ] 角色权限模型（Owner / Editor / Viewer）
- [ ] 文章分享链接
- [ ] 多人协同编辑（Yjs）
- [ ] 团队空间

### Week 5-6：图片 CDN

- [ ] 统一图片上传到自有 CDN
- [ ] 发布时自动替换图片 URL
- [ ] 图片缩略图 / 裁剪处理
- [ ] 避免各平台图片外链失效

**Phase 3 交付物**：

- 用户注册登录系统
- 文章云端存储，跨设备同步
- 多人协同编辑
- 自有图片 CDN

## Phase 4：模板与样式（预计 4-6 周）

**目标**：平台专属 CSS 模板，从 Markdown 升级为类 H5 富文本

### 微信公众号 H5 渲染

- [ ] 代码块语法高亮 → 渲染为图片
- [ ] 表格窄屏自适应
- [ ] 封面图自动裁剪（2.35:1）
- [ ] 正文图片自动转存微信 CDN
- [ ] 段落间距 / 字号 / 行高自动适配微信客户端

### 小红书图文模板

- [ ] 多图排版（4/6/9 图布局）
- [ ] 正文 emoji 分段点缀
- [ ] 封面图自动裁剪（3:4）
- [ ] 话题标签智能排序

### 通用模板引擎

- [ ] 模板 DSL（JSON 定义模板参数）
- [ ] 预设模板库
- [ ] 品牌自定义（Logo / 配色 / 字体）
- [ ] CSS 样式引擎：AST → 内联样式 HTML

**Phase 4 交付物**：

- 微信公众号 H5 级富文本渲染
- 小红书多图排版模板
- 可扩展的模板引擎

## 技术债务管理

以下事项随 Phase 推进逐步处理：

- 服务端存储（IndexedDB → PostgreSQL）：Phase 3
- 图片 CDN（原始 URL → 自有 CDN）：Phase 3
- 单元测试覆盖：Phase 6
- 错误监控（Sentry）：Phase 6
- 性能优化（代码分割、懒加载）：持续
